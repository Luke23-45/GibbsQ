
\section{N-GibbsQ: Learned Neural Policy}
\label{sec:neural_policy}

The $\ngibbsq$ policy is a learned routing law that uses a neural network to map queue states to routing distributions.
It is an \emph{empirical} approximation layer and is not part of the theorem statements.

\subsection{Architecture}

The actor network takes as input the queue-length vector $\Qstate \in \Nats^\Nservers$ and service rates $\srvrate \in \Reals_+^\Nservers$.
Queue lengths are preprocessed via $\log(1 + \Qlen{i})$ normalization to stabilize input magnitudes.
A feedforward network with two hidden layers and ReLU activations produces routing logits, which are passed through a softmax to yield
\begin{equation}
\label{eq:neural_output}
\rprob{i}(\Qstate, \srvrate; \theta) = \softmax\!\bigl(f_\theta(\Qstate, \srvrate)\bigr)_i,
\end{equation}
a valid probability distribution on $\Simplex$ by construction.

\subsection{Two-Phase Training Pipeline}

\subsubsection{Phase~1: Behavior Cloning}
The network is trained via cross-entropy loss against an analytical teacher policy.
The default teacher is UAS with $\temp = 10$; the ablation study (Section~\ref{sec:exp:ablation}) also evaluates Calibrated UAS as an alternative teacher.
Training runs for 1{,}000 steps, reaching 97--99\% top-1 accuracy.
The BC phase provides a stable initialization that prevents catastrophic divergence during Phase~2.

\subsubsection{Phase~2: REINFORCE Fine-Tuning}
Starting from the BC-pretrained weights, we apply policy gradient optimization using \reinforce{} with a running-average baseline~\cite{williams1992simple}.
Training uses 16-episode batches over 15 epochs, with each episode simulating 1{,}000 time units of queue dynamics.
The gradient estimator's directional correctness is verified in Appendix~\ref{sec:appendix_gradient} (cosine similarity $= 1.0$).
Full hyperparameter details are provided in Appendix~\ref{sec:appendix_training}.
